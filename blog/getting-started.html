Getting Started: How I Set Up This Blog

A quick walkthrough of this static blog setup and how to add new posts — no build tools required.

Why a static blog?

This site is built with plain HTML, CSS, and JavaScript — no frameworks, no build step. That makes adding a blog section straightforward: each post is just an HTML file in the blog/ folder.

How to write a new post

Follow these steps each time you want to publish something:

  1. Copy this file (blog/getting-started.html) and rename it, e.g. blog/my-new-post.html.
  2. Update the <title>, <meta name="description">, and the <h1> inside the article.
  3. Change the <time datetime="..."> to today's date.
  4. Replace the tags (<span class="badge">) with your own topics.
  5. Write your content inside the <div class="blog-body"> section — use standard HTML: <h2>, <p>, <ul>, <pre>, <code>, etc.
  6. Open blog/index.html and add a new <article> card at the top of the list, linking to your new file.

Useful HTML snippets for writing

Code block:

<pre><code>
SELECT user_id, COUNT(*) AS events
FROM events
GROUP BY user_id
ORDER BY events DESC;
</code></pre>

Inline code: wrap text in <code> tags.

Callout / notice box:

<div class="notice">
  Key insight: always validate before writing to production.
</div>

Which renders as:

Key insight: always validate before writing to production.

That's it

No npm install. No build command. Edit HTML, commit, push — GitHub Pages serves it instantly.
